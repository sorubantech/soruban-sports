## 1. Stage Structure (Proposed)

Like school grades, a student progresses through stages. Each stage has defined skills to learn, practice, and pass.

### Proposed 7-Stage Model

| Stage | Name | Age Group (Suggested) | Focus |
|-------|------|----------------------|-------|
| **Stage 1** | Foundation | 6–8 yrs | Basic grip, stance, hand-eye coordination, fun drills |
| **Stage 2** | Beginner | 8–10 yrs | Basic batting & bowling technique, catching, throwing |
| **Stage 3** | Developing | 10–12 yrs | Shot selection, bowling variations, fielding positions |
| **Stage 4** | Intermediate | 12–14 yrs | Match awareness, advanced shots, spin/pace basics |
| **Stage 5** | Advanced | 14–16 yrs | Role specialization, match strategy, fitness regime |
| **Stage 6** | Elite | 16–18 yrs | Tournament readiness, mental conditioning, advanced analytics |
| **Stage 7** | Professional Prep | 18+ yrs | Selection trials prep, first-class cricket readiness |

> **Note:** Age is a guideline, not a rule. A talented 10-year-old could be in Stage 4. Progression is skill-based, not age-based.

> **Academy Freedom:** Academies can rename stages, add/remove stages, or adjust the mapping. The platform provides a default template.

## 2. Skill Categories (Subjects)

Each stage covers multiple skill categories, just like subjects in a school.

### Primary Skill Categories

| Category | Sub-Categories |
|----------|---------------|
| **Batting** | Grip, Stance, Backlift, Front foot shots, Back foot shots, Sweep shots, Unorthodox shots, Running between wickets |
| **Bowling** | Fast bowling, Spin bowling (off-spin, leg-spin, left-arm), Run-up, Action, Variations, Line & length |
| **Fielding** | Catching (close, outfield), Ground fielding, Throwing (direct hit, relay), Sliding, Boundary saves |
| **Wicketkeeping** | Stance, Taking to pace, Taking to spin, Stumping, Catches, Footwork |
| **Fitness** | Endurance, Speed & agility, Strength, Flexibility, Injury prevention |
| **Mental / Game Sense** | Match awareness, Pressure handling, Shot selection under situations, Captaincy basics, Communication |

## 4. How Curriculum Builder Works

### Platform Provides (Common Library)
- Animated videos for every shot/skill listed above
- Created by cricket experts (ex-international players, certified coaches)
- Tagged by: skill category, stage, player role, difficulty
- Includes: slow-motion breakdown, common mistakes, drill instructions

### Academy Adds (Custom Content)
- Their own coaching videos
- Academy-specific drills
- Local training exercises
- Notes, PDFs, or reference material

### Combined Curriculum (What Students See)
```
Stage 3 — Developing — Batting
├── Cover Drive
│   ├── [Platform] Animated Video: "Cover Drive Technique"
│   ├── [Platform] Common Mistakes Video
│   ├── [Platform] Drill: Cone Target Cover Drive
│   ├── [Academy] Coach Ravi's Cover Drive Demo (uploaded video)
│   └── [Academy] Our Academy's Foot Movement Drill
├── Pull Shot
│   ├── [Platform] Animated Video: "Pull Shot"
│   ├── [Platform] Drill: Short Ball Pull
│   └── [Academy] Custom: Tennis Ball Pull Drill for Juniors
├── ...
```

### Academy Controls
- **Include / Exclude** any platform material per stage
- **Reorder** skills within a stage
- **Add prerequisites** — e.g., "Must complete Front Foot Defence before Cover Drive"
- **Set passing criteria** — e.g., "8/15 in drill to pass" or "Coach approval"
- **Create custom stages** — e.g., "IPL Masterclass" as a special Stage 5.5

## 5. Role-Based Curriculum Path

Not every student follows the same path. Based on their role:

### Batsman Path
All batting stages in full depth + Fielding + Fitness + Game Sense

### Bowler Path (Fast / Spin)
All bowling stages in full depth + Batting basics (Stages 1-3) + Fielding + Fitness + Game Sense

### Allrounder Path
Full batting path + Full bowling path (one type) + Fielding + Fitness + Game Sense

### Wicketkeeper Path
All wicketkeeping stages + Batting in full depth + Fielding + Fitness + Game Sense

> **Flexibility:** Academy decides the balance. Some may want all students to go deep in batting regardless of role.

## 6. Assessment / Exam System

### Types of Assessment

| Type | Description | When |
|------|-------------|------|
| **Skill Test** | Test a specific skill (e.g., "Cover drive drill — 15 balls") | After completing skill material |
| **Stage Exam** | Comprehensive test of all skills in a stage | End of stage |
| **Match Simulation** | Simulated match scenarios with scoring | Stage 4 onwards |
| **Video Assessment** | Student uploads batting/bowling video, AI + coach review | Any time |
| **Fitness Test** | Physical fitness benchmarks (sprint time, endurance, strength) | Every stage |

### Scoring Model (Proposed)
```
Skill Score     = Drill performance (60%) + Coach assessment (40%)
Stage Score     = Average of all skill scores in that stage
Overall Rating  = Weighted average across stages completed
```

### Passing Criteria
- Academy sets minimum passing score per stage (e.g., 65%)
- Must pass all mandatory skills in a stage to progress
- Optional skills can be skipped but shown in progress report
- Failed? → System recommends specific drills to retry

## 8. Decisions Made (2026-03-16)

| # | Question | Decision | Notes |
|---|----------|----------|-------|
| 1 | How many stages? | **7 Stages — Configurable** | Platform provides 7 as default template. Academies can add/remove/rename stages. Fully configurable. |
| 2 | Time-bound stages? | **No — Purely skill-based** | No minimum duration. A fast learner can progress in weeks, a slower one takes months. Platform tracks time spent per stage for analytics only. |
| 3 | Who decides stage promotion? | **Coach must approve** | Even if student passes all exams, final promotion requires coach approval. System notifies coach when student is eligible. Coach can approve or hold with a reason. |
| 4 | Pre-assessment at admission? | **Yes — Coach-driven placement** | No automated test at admission. Coach evaluates the student (trial session, observation) and places them directly into the appropriate stage. A Stage 1 student and a Stage 3 student can both join on the same day. |
| 5 | Digital certificates? | **Yes** | Students receive a digital certificate on completing each stage. Shareable (PDF/image). Builds parent trust and student motivation. Academy branding on certificate + platform branding. |
| 6 | Cross-academy portability? | **No** | If a student switches academies, they start fresh at the new academy. Each academy has its own curriculum and standards — progress is not transferable. Historical data stays with the old academy. |

## 9. BA Recommendations — Additional Decisions Needed

### 9.1 Stages — Default Template Rationale

The 7-stage model maps well to the real cricket development journey in India:

```
Stage 1 (Foundation)      → "Come play cricket" — fun, basic motor skills
Stage 2 (Beginner)        → "Learn the basics" — textbook technique
Stage 3 (Developing)      → "Build your game" — shot repertoire
Stage 4 (Intermediate)    → "Think cricket" — match situations, strategy
Stage 5 (Advanced)        → "Specialize" — role mastery, high intensity
Stage 6 (Elite)           → "Compete" — tournament/selection level
Stage 7 (Professional Prep) → "Go professional" — first-class readiness
```

This covers the full spectrum from a 6-year-old picking up a bat to an 18-year-old preparing for Ranji Trophy trials. Academies that only serve juniors (ages 6-12) might use Stages 1-3 only. Elite academies might start from Stage 4.

### 9.2 Coach Approval Workflow (Recommended)

```
Student completes all mandatory skills in Stage 3
        ↓
System status: "Eligible for promotion"
        ↓
Notification sent to assigned coach
        ↓
Coach reviews: exam scores + attendance + session observations
        ↓
    ┌─────────┐          ┌──────────┐
    │ APPROVE │          │   HOLD   │
    └────┬────┘          └────┬─────┘
         ↓                    ↓
  Student promoted       Coach adds reason:
  to Stage 4             "Needs more match
  Certificate generated  practice before
  Parent notified        moving up"
                              ↓
                         Student sees:
                         "Focus areas before
                         next stage" + recommended
                         drills
```

### 9.3 Admission & Placement Flow (Recommended)

```
Student/Parent applies to academy on platform
        ↓
Academy receives application (with basic info: name, age, experience)
        ↓
Academy schedules a trial session (optional — academy decides)
        ↓
Coach observes student: batting, bowling, fielding, fitness
        ↓
Coach assigns:
  ├── Role: Batsman / Bowler / Allrounder / Wicketkeeper
  ├── Stage: Places into appropriate stage (e.g., Stage 3)
  └── Batch: Assigns to a training batch/group
        ↓
Student onboarded → Sees their Stage 3 curriculum immediately
Parent notified with placement details
```

### 9.4 Certificate Design Elements (Recommended)

Each stage completion certificate should include:
- **Student name & photo**
- **Academy name & logo**
- **Platform branding** (Soruban Sports — small, bottom corner)
- **Stage completed** (e.g., "Stage 3 — Developing")
- **Role** (e.g., "Batsman")
- **Key scores:** Top 3 skills with scores
- **Coach name** who approved promotion
- **Date of completion**
- **QR code** linking to student's public profile (optional — academy can enable/disable)
- **Certificate ID** for verification

Format: Downloadable PDF + shareable image (for WhatsApp/Instagram)

### 9.5 Data on Academy Switch (No Portability)

When a student leaves an academy:
- Academy retains all historical data (attendance, scores, assessments)
- Student's app shows "Past Academy: XYZ (inactive)"
- Student **cannot** access old academy's curriculum or materials
- New academy places student fresh through their own admission process
- **Reason:** Each academy has different curriculum, standards, and coaching philosophy. Carrying over a "Stage 4" from one academy means nothing at another.
